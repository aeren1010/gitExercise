Über uns

Unterschiedliche Interessen, ein gemeinsames Projekt

Wir sind ein Team aus Schülerinnen und Schülern, das im Rahmen eines Praktikums eine gemeinsame Website entwickelt. Dabei verbinden wir individuelle Profile mit praktischen Grundlagen in HTML, CSS, Git und GitHub.

Das Projekt

Ziel der Team Presentation Page

Die Website soll die einzelnen Teammitglieder übersichtlich vorstellen und gleichzeitig zeigen, wie mehrere Personen gemeinsam an einem digitalen Projekt arbeiten können.

01

Gemeinsame Struktur

Alle Seiten verwenden dieselbe Navigation, denselben Header, denselben Footer und ein zentrales Stylesheet.

02

Individuelle Profile

Jedes Teammitglied erhält eine eigene Seite für Interessen, Erfahrungen und Kompetenzen.

03

Versionskontrolle

Git speichert die einzelnen Entwicklungsstände. GitHub ermöglicht die Zusammenarbeit und Veröffentlichung.

Zusammenarbeit

Aufteilung der Aufgaben

1

Aufgabe auswählen

Eine Person übernimmt beispielsweise die Startseite, das Design oder eine eigene Profilseite.

2

Git-Branch erstellen

Die Änderung wird getrennt vom Hauptstand entwickelt, damit andere Teammitglieder parallel arbeiten können.

3

Änderung prüfen

Nach dem Commit und Push wird ein Pull Request erstellt und von einem anderen Teammitglied kontrolliert.

4

Änderungen zusammenführen

Nach erfolgreicher Prüfung wird der Branch mit dem Hauptbranch zusammengeführt.
